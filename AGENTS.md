# AGENTS.md

## Alcance

Estas instrucciones aplican a todo el proyecto `Los mecanismos invisibles`.

## Proyecto

Sitio web vanilla con Vite para un medio editorial de pensamiento. La identidad central debe seguir el `README.md`.

## Comandos

```bash
npm install
npm run dev
npm run build
```

No agregar dependencias nuevas salvo que sean necesarias para la tarea.

## Direccion editorial

- Mantener una voz sobria, clara, elegante, reflexiva y precisa.
- Priorizar comprension antes que opinion.
- Evitar tono motivacional, autoayuda, polemica artificial o frases hechas.
- Cada pieza debe defender una sola idea y responder una pregunta central.
- La experiencia concreta funciona como laboratorio; el objetivo real es revelar el mecanismo.
- No inventar ensayos, titulos, preguntas, fragmentos ni contenidos editoriales. Usar solo textos reales ubicados en `ensayos/`.

## Direccion visual

- Usar una estetica limpia, silenciosa, editorial y atemporal.
- Priorizar marfil, blanco roto, crema, arena, lino, beige claro, gris piedra y verde oliva muy desaturado.
- Usar dorado envejecido, bronce claro y gris grafito solo como apoyo.
- Evitar negro absoluto como identidad principal, rojo intenso, azul electrico, verdes saturados, neones y gradientes llamativos.
- Usar mucho espacio en blanco, pocos elementos graficos y jerarquia tipografica marcada.
- Preferir titulos con serif clasica y textos secundarios con sans serif limpia.

## Implementacion

- Modificar solo lo necesario para la tarea.
- Preservar la arquitectura vanilla/Vite salvo instruccion contraria.
- No exponer secretos ni credenciales.
- Usar variables de entorno para cualquier credencial futura.
- Antes de cambiar dependencias, explicar la razon.
- Verificar con `npm run build` cuando se modifique codigo de produccion.
